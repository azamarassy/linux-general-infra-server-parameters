# Server Configuration Parameter Sheets

このリポジトリは、**Rocky Linux 8** をベースとした汎用的なインフラサーバー構築における詳細なシステム設定値（パラメーターシート）を管理・ドキュメント化したものです。

構築時の構成管理、セキュリティ監査、およびトラブルシューティング時の参照用として活用することを目的としています。

## プロジェクトの目的

* **構成の可視化**: サーバー内部の複雑な設定を一覧化し、属人化を防止。
* **セキュリティ基準の準拠**: CISベンチマークに基づいた推奨値と実設定の比較。
* **保守性の向上**: 各項目の確認コマンドを併記し、実機での検証を容易にする。

## ドキュメント構成

本リポジトリのパラメーターシートは、管理対象ごとに以下の3つのディレクトリに分かれています。

### 1. 01_system（システム基本構成）

サーバーの土台となるOS設定およびシステム基盤に関する定義です。

* **基本設計 (basic_settings)**: ホスト名、言語、カーネル等の基本パラメータ
* **chrony**: NTPサーバー設定および時刻同期ポリシー
* **dnf**: パッケージ管理のリポジトリ・自動更新設定
* **fstab**: ディスクマウント構成およびUUID定義
* **ログローテート (log_rotate)**: ログの世代管理・圧縮設定
* **swap**: スワップ領域のサイズおよび有効化構成

### 2. 02_security（アカウント・認証基盤）

ユーザー管理とアクセス制御に関するセキュリティ定義です。

* **アカウントポリシー (account_policy)**: パスワード強度・ロックアウト規定
* **グループ (group)**: システム内グループの一覧と所属定義
* **セキュリティ (security)**: PAM認証等のシステム認証基盤設定
* **sudo**: 役割に応じた特権実行の制限・許可設定
* **ユーザー (user)**: ユーザー属性、UID、権限の定義

### 3. 03_services（サービス・ログ収集）

特定のネットワークサービスやログ管理エンジンに関する定義です。

* **rsyslog**: システムログの出力ルールおよびモジュール設定
* **ssh**: SSHサーバーのセキュリティ・接続制限設定

## 免責事項

本リポジトリに含まれる設定値はデフォルトまたは標準的な推奨値に基づいています。実際の商用環境に適用する場合は、各システムの要件に合わせて適切に調整してください。
